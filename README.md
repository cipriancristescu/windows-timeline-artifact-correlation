# Windows Timeline Artifact Correlation

This repository contains the source code used for the bachelor thesis **“Corelarea artefactelor digitale pentru reconstrucția activității unui sistem Windows”**.

The application reconstructs Windows activity timelines by parsing and correlating exported forensic artifacts from three sources:

- Browser History;
- Windows Prefetch;
- Master File Table (MFT).

The tool normalizes heterogeneous CSV exports into a common event model, filters low-value system noise, groups events into activity sessions, and applies deterministic cross-artifact correlation rules. It supports both a command-line workflow and a Streamlit interface.

## Features

- Parses Browser History, Windows Prefetch, and MFT CSV exports.
- Normalizes records from different sources into a common event model.
- Filters irrelevant or noisy system events.
- Groups events into activity sessions using temporal and semantic rules.
- Assigns confidence levels to groups based on the available sources.
- Applies deterministic correlation rules across artifacts.
- Provides a Streamlit interface for interactive analysis.
- Exports results for reporting.
- Includes automated tests for parsers, normalization, filtering, grouping, correlation, and export behavior.
- Supports optional local AI-assisted interpretation through Ollama.

## Experimental Environment

The experiments described in the thesis were executed on a controlled Windows 10 virtual machine.

Virtual machine configuration:

```text
Operating system: Windows 10, 64-bit
Virtualization: Oracle VM VirtualBox
RAM: 4096 MB
Virtual CPUs: 2
Virtual disk: 50 GB
Network: NAT
```

The artifact exports used for testing were generated by executing controlled user activities inside this virtual machine and exporting the resulting data as CSV files.

## Controlled Activities

The dataset was generated from the following controlled scenarios:

- web browsing;
- downloading and opening PDF documents;
- local file operations;
- creating, editing, and running a Python script;
- archive creation and extraction;
- exporting forensic artifacts.

These activities were chosen to generate observable traces across one or more of the analyzed artifact sources.

## Setup

Create a Python virtual environment:

```powershell
python -m venv .venv
```

Activate it:

```powershell
.venv\Scripts\activate
```

Install the required dependencies:

```powershell
pip install -r requirements.txt
```

## Running the Streamlit Interface

Start the Streamlit application:

```powershell
streamlit run app.py
```

Then open the local URL displayed by Streamlit.

The interface allows selecting the exported CSV files, configuring filters, running the analysis, inspecting activity groups, and exporting results.

## Running the CLI

The project also provides a command-line entry point:

```powershell
python main.py --input path\to\input --output path\to\output
```

The input directory should contain the exported artifact CSV files expected by the parsers.

If the local argument names differ, check the available options with:

```powershell
python main.py --help
```

## Running Tests

Run the automated tests with:

```powershell
pytest
```

The tests cover core deterministic components such as parsing, normalization, filtering, grouping, correlation, and export behavior.

## Reproducing the Thesis Experiment

To reproduce the demonstration workflow:

1. Clone this repository.
2. Create and activate a Python virtual environment.
3. Install the dependencies from `requirements.txt`.
4. Request access to the dataset archive linked above.
5. Download and extract the dataset.
6. Run the Streamlit interface or the CLI pipeline.
7. Use the extracted CSV files as input.
8. Review the generated activity groups and correlation findings.

The expected high-level result is a timeline containing activity groups related to:

- web browsing;
- PDF download and opening;
- local file operations;
- Python script development and execution;
- archive creation/extraction and CSV export.

The deterministic pipeline should produce reproducible results when the same input files and configuration are used.

## Outputs

Generated outputs should be written outside the tracked source tree or inside an ignored output directory.

Typical outputs include:

```text
timeline.csv
timeline.json
timeline_groups.json
PDF report
```

These files are generated locally and are not tracked in this repository by default.

## Configuration

Default settings are stored in:

```text
config/app_config.yaml
```

The configuration controls parser behavior, filtering rules, grouping thresholds, export options, and optional AI analysis settings.

## Optional AI Analysis

The deterministic pipeline does not require an AI model.

If Ollama is installed locally and a compatible model is available, the application can generate an additional natural-language interpretation for selected activity groups.

This interpretation is auxiliary and does not affect:

- event parsing;
- filtering;
- grouping;
- confidence levels;
- deterministic correlation findings.
